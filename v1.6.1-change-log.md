# v1.6.1 请求头/请求体白名单制 + 图片格式归一化

## 改动日期
2026-07-02

## 改动文件
- `src/modules/proxy_server.py`（Windows + Mac 同步）
- `src/ui/pages/api_proxy.py`（Windows + Mac 同步）

## 改动内容

### 1. 请求头：白名单制
**位置**：主请求 L1924-1937，摘要请求 L2258-2266

| 旧 | 新 |
|----|-----|
| Content-Type | Content-Type |
| Authorization | Authorization |
| X-Request-ID（trace头） | Accept: application/json, text/event-stream |

去掉 `X-Request-ID`（trace 头，不需要发给上游），新增 `Accept`。

### 2. 请求体：白名单制
**位置**：L1947-1960

- **旧**：原样透传 `request_data`（客户端发啥转啥）
- **新**：白名单过滤，只保留上游已知接受的字段，其余删除并记日志

白名单字段：
```
model, messages, stream, stream_options,
temperature, top_p, max_tokens, presence_penalty,
frequency_penalty, stop, tools, tool_choice, response_format,
parallel_tool_calls, seed, user, metadata, logprobs, top_logprobs, n
```
（v1.6.1-fix 补充了后7个 OpenAI 兼容常用字段）

被删除的字段日志：`[v1.6.1] 请求体白名单过滤，删除字段: [...]`

### 3. messages 图片格式归一化
**位置**：L1962-1965（调用处）+ L238-296（新函数 `_normalize_messages_for_upstream`）

- **旧**：原样透传 messages（WorkBuddy 发 `input_image`，代理转 `input_image` 给上游 → 上游纯 API 模式不认 → 图片被丢弃）
- **新**：转发前归一化

| 原始格式 | 归一化后 |
|---------|---------|
| `{type: "image_url", image_url: {url: "data:..."}}` | 保持不变 |
| `{type: "input_image", image: {data: base64, mediaType: mime}}` | `{type: "image_url", image_url: {url: "data:mime;base64,..."}}` |
| `{type: "input_image", image: "data:image/png;base64,..."}` | `{type: "image_url", image_url: {url: "data:..."}}` |
| `{type: "input_image", image: {url: "data:..."}}` | `{type: "image_url", image_url: {url: "data:..."}}` |
| `{type: "image", ...}` | 同 input_image |
| `{type: "image_blob_ref", ...}` | `{type: "text", text: "[图片引用无法解析]"}`（blob在客户端本地） |
| 其他 type | 保持不变 |

日志新增 `img_normalized=N` 字段统计归一化数量。

### 4. [v1.6.1-fix] model 缺失写回 request_data
**位置**：L1822-1824

- **旧**：`model = request_data.get("model", "")` + `if not model: model = "auto"` — 只改局部变量
- **新**：补充 `request_data["model"] = model`，确保白名单转发时上游能收到 model 字段

### 5. [v1.6.1-fix] messages 类型严格校验
**位置**：L1827-1830

- **旧**：`messages = request_data.get("messages", [])` + `if len(messages) < 2` — 如果 messages 是字符串/None/dict 会崩溃
- **新**：先校验 `isinstance(messages, list) and messages`，不合法直接返回 400

### 6. [v1.6.1-fix] CORS 预检入站头放宽
**位置**：L1750 (do_OPTIONS)

- **旧**：`Access-Control-Allow-Headers: Authorization, Content-Type, Accept` — WorkBuddy 预检带 X-Product-Version 会被浏览器层挡住
- **新**：`Access-Control-Allow-Headers: *` — 入站允许任意头，出站仍由白名单丢弃

### 7. [v1.6.1-fix] UI 状态检测请求头统一
**位置**：`src/ui/pages/api_proxy.py` L1427-1431

- **旧**：`Content-Type + Authorization + X-Request-ID: check_status_001`
- **新**：`Content-Type + Accept + Authorization`（去掉 X-Request-ID，补 Accept，与主链路一致）
