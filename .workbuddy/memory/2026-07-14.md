## 修复仪表盘 UI: 缺失日期补全

**问题**: 用户两天没用 → 数据库无记录 → 日历热力图无数据格子的位置"完全失效"，折线图最后一段没数据点。

**修复**:
1. **后端 `src/modules/daily_stats.py: get_calendar()`**: 改用 `dt.date` 显式计算起始日期 (months 个月的 1 号)，按日循环补全缺失日期为 `{credits: 0, count: 0}`。返回最近 N 个月的完整日期序列。
2. **前端 `web/static/index.html`**:
   - 日历格子新增 `isFuture` 标志，未来日期淡化显示 (text-outline/40)，过去/今天的"无消耗"格子用更清晰的 secondary 文本色显示 0
   - 无消耗格子改用 `bg-surface-container-lowest` 替代 `bg-white`（避免深色模式撞色）
   - 折线图新增 `trendAllZero` computed：30 天全是 0 时叠加"最近30天无消耗"提示

**验证**: 模拟 4 个月数据，中间穿插缺失日期，后端正确返回 105 天连续日期（首尾两端含 0）。

## 趋势折线图：鼠标缩放 + 平移

**改动**: `web/static/index.html` 趋势图 JS 部分重写（模板+逻辑）：
- **状态**: `trendZoom`(1~10)、`trendPan`(data point offset)、`trendWindowSize`(visible point count)
- **滚轮缩放**: `@wheel.prevent` → zoom ×1.12(缩小)/×0.9(放大)，保持中心点稳定；缩放范围 1x(30天) ~ 10x(3天)
- **鼠标拖拽平移**: `@mousedown/mousemove/mouseup` 全局监听，鼠标偏移换算数据点偏移；cursor 切换 grab↔grabbing
- **UI 提示**: 缩放时标题栏显示"视窗: N天" + 重置按钮，点击恢复 1x 全览
- **X 轴标签**: 缩放模式下(≤14天)全显示日期标签，避免空白
- **Y 轴标签**: 新增左侧 Y 轴刻度数值
- **数据切片**: `drawTrendChart()` 改为按 zoom/pan 窗口从 30 天数据 slice，窗口最小 3 天

**踩坑**: 模板里 `id="trendWheelZone"` + `id="trendContainer"` 重复，HTML 只保留第一个，导致 JS 里 `getElementById('trendContainer')` 返回 null，`drawTrendChart()` 直接 return，图表不渲染。删掉重复 id 即可。**教训：给已有元素加 id 前先检查现有 id，DOM 不会警告但行为会静默失效。**
