# 代理 Key 自动禁用问题 — 根因分析

**日期**：2026-07-18

## 现象

代理面板中，Key 会被自动禁用，且通过 UI 启用后重启服务又变回禁用状态。

## 根因：两个 Bug 叠加

### Bug 1：update_upstream_key / delete_upstream_key 匹配字段不一致

**位置**：src/modules/proxy_server.py → ProxyDatabase.update_upstream_key() / delete_upstream_key()

- 导入 Key 时，内部 key_id 是随机生成的（如 ck_a1b2c3d4），真正的密钥存 pi_key 字段（如 ck_831a5186）
- toggle 接口查找 Key 时同时匹配 key_id 和 pi_key（能查到）
- 但 update_upstream_key 只匹配 key_id（写不到）
- **结果**：toggle 接口返回 {"success": true} 但实际未写入，Key 状态从未变更

**修复**：两个方法都改为同时匹配 key_id 和 pi_key：

`python
# update_upstream_key
if k.get("key_id") == key_id or k.get("api_key") == key_id:

# delete_upstream_key
if k.get("key_id") != key_id and k.get("api_key") != key_id
`

### Bug 2：toggle 接口逻辑反了

**位置**：web/api/proxy.py → proxy_toggle_key() 第 137 行

原逻辑：

`python
new_status = "disabled" if current in ("active", "exhausted", "cooldown", "rate_limited") else "active"
`

- 当 Key 状态是 exhausted（已耗尽）或 cooldown（冷却中）时，点 toggle 期望的是「启用」
- 但实际执行的是把它设成 disabled（已禁用）
- 只有 disabled / permanent_disabled / bnormal 这些状态点 toggle 才会变 active

**修复**：简化为二元翻转：

`python
new_status = "disabled" if current == "active" else "active"
`

现在逻辑：active → disabled，其他任何状态 → active。
