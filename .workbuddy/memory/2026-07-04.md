## BugFix: 导出/导入字段名不匹配 + 旧 Key 无 key_id 404

### 修复的文件
- `web/api/accounts.py`：ImportAccountItem 加 `auth_token` 字段；import 处统一用 `access_token = item.access_token or item.auth_token`；export 同时输出 `access_token` 和 `auth_token`
- `web/api/proxy.py`：`proxy_status` 的 `id` 回退到 `api_key`（旧 key 无 key_id）；delete/toggle 用 `k.get("key_id") == key_id or k.get("api_key") == key_id`
- `src/modules/proxy_server.py`：新增 `_find_key_index()` 同时匹配 `key_id` 和 `api_key`；`update_upstream_key`/`delete_upstream_key` 改用此方法
- `web/static/index.html`：导入解析 `filter()` 加 `it.auth_token`；`map()` 中 `access_token: it.access_token || it.auth_token`

### 根因
1. **导出/导入格式不匹配**：export 返回 `auth_token`，import filter 只检查 `access_token`，导致 JWT 账号被过滤
2. **旧 Key 无 key_id**：旧版 `addProxyKey` 不设 `key_id` → status 返回 `id=""` → URL `keys//delete` → 404；DB 方法只按 `key_id` 匹配

### 问题识别
- Web UI `web/static/index.html` 的 "添加代理密钥" 弹窗（第 485-514 行）是纯手动输入 `api_key` + `account_name`
- 缺少与账号系统的集成，用户需重复输入已有的 API Key
- 桌面 UI `src/ui/pages/api_proxy.py:1220-1261` 已有完整 `_import_from_accounts()` 实现——从账号自动提取 `api_key`/`auth_token`，带标签和积分

## BugFix: Web 端 autoStartProxy 设置不生效

### 根因
`autoStartProxy` 设置的读写链路完整（HTML 开关 → API → 持久化存储），但启动时没有代码消费该设置。

### 修复
- `web/server.py`：`main()` 中 `init_db()` 后检查 `load_setting("autoStartProxy")`，若为 `True` 则调用 `proxy_start()` 自动启动代理
- 默认值为 `True`，首次启动（无历史设置）也会自动启动
