## Bug Fix: 配额详情套餐数显示 0（缓存 packages 字段残缺）

**根因：** proxy_server.py 中 `sync_quota_to_key` 保存的 packages 只含 4 个字段（cycle_remain, cycle_end, package_name, package_type），缺少 `cycle_size`。而 proxy.py 渲染时要求 `cycle_size > 0` 才显示，导致所有套餐被过滤。

**时间线：**
- `cache=false`（手动刷新）→ 写完整 8 字段 packages 到 DB → 显示正常
- 任何 Key 请求后 5 分钟 → `refresh_key_points_if_needed` 触发 → 写残缺 4 字段 packages 覆盖 DB → 第二天 `cache=true` 读到残缺版 → 全显示 0

**修改文件：**
1. `src/modules/proxy_server.py` — 三处补全字段：
   - `refresh_key_points_if_needed:1648-1653` — pkgs 构造时补 cycle_size, cycle_start
   - `sync_quota_to_key:1296-1322` — dict 分支和 ResourcePackage 对象分支均补 cycle_size, cycle_start
2. `web/static/index.html:1394-1404` — `loadQuotaCache` 增加 packages 有效性判断：服务器 packages 全空时不直接 return，fall through 到 localStorage 兜底

## Bug Fix: NameError: allowed_key_ids is not defined（生产环境崩溃）

**根因：** upstream commit `08fad3d`（qinchangxv, 2026-06-24）在 `_handle_chat_completions` 中使用 `allowed_key_ids` 变量但从未从 `sub_key` 提取。同函数内 `allowed_models` 和 `key_mode` 都有 `sub_key.get(...)` 提取，唯独 `allowed_key_ids` 漏了。

**影响：** 每次请求 `/v1/chat/completions` 都会 `NameError` 崩溃，服务完全不可用。

**修改文件：**
- `src/modules/proxy_server.py:2855` — 在 while 循环前添加 `allowed_key_ids = sub_key.get("allowed_key_ids", [])`
